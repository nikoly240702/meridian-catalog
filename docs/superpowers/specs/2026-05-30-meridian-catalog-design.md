# ТД «Меридиан» — сайт-каталог. Дизайн-спецификация

**Дата:** 2026-05-30
**Статус:** утверждён к реализации

## 1. Назначение и контекст

Презентационный сайт-каталог для торгового дома «Меридиан». Создаётся в рамках развития
проекта Tender Sniper: компания выходит с тендеров на прямые контракты с заказчиками, и для
заключения прямых соглашений нужен сайт-каталог, который выглядит как солидный действующий
поставщик.

**Главная цель:** производить впечатление надёжного работающего поставщика, чтобы сайт можно
было показывать потенциальным заказчикам.

**Чем НЕ является:**
- Не интернет-магазин (нет корзины, цен, оплаты).
- Не инструмент сбора заявок (формы — визуальные заглушки, бэкенда нет).
- Не реальный товарный каталог (товары, фото и реквизиты — демо-заглушки).

Модель бизнеса: товара на складе нет, поставки организуются под заказ. Поэтому везде —
«Цена по запросу» и метка «Под заказ», а не цены и наличие.

## 2. Бренд

- **Название:** ТД «Меридиан» (торговый дом).
- **Слоган:** «Поставки для бизнеса и госзаказа».
- **Логотип:** знак-«меридиан» — окружность с вертикальной красной осью — плюс название
  набором Oswald. В подзаголовке логотипа: «ТОРГОВЫЙ ДОМ · СНАБЖЕНИЕ».
- Позиционирование: универсальный поставщик промышленного снабжения (по образцу
  promkomplex.ru и dalkos.ru), работающий с 44-ФЗ / 223-ФЗ и прямыми договорами.

## 3. Визуальный язык (утверждён на главной странице)

- **Палитра:** чёрный `#0E0F12`, тёмно-серый `#16181D`/`#23262D`, красный акцент `#E11428`
  (наведение `#B60E1F`), светлый фон `#F2F3F5`, белый `#FFFFFF`, текст `#15171C`.
- **Шрифты:** Oswald (500–700, капс) — заголовки, кнопки, цифры; Manrope (400–700) —
  основной текст. Подключение — Google Fonts.
- **Стиль фото:** ч/б (grayscale) с затемнением/градиентом для цельной индустриальной подачи.
  В демо используются тематические заглушки по ключевым словам; на боевом сайте заменяются
  реальными фото товара/категорий.
- **Подача:** плотная, «индустриальная», крупные заголовки капсом, красные акцентные плашки,
  hover-инверсии и зум на карточках.
- **Адаптив:** обязателен (десктоп / планшет / мобильный). Брейкпоинты ~1040px и ~600px.

## 4. Архитектура

Статический сайт: чистые HTML + CSS + JS, без фреймворков, без сборщика, без бэкенда.

- Общие визуальные части (шапка top-bar + header, подвал, общий CSS) повторяются на каждой
  странице. CSS вынесен в общий файл `assets/styles.css`; страницы подключают его.
- Никаких внешних зависимостей кроме Google Fonts и демо-изображений.
- Сайт открывается двойным кликом по `index.html` и так же легко размещается на любом
  хостинге (статика).
- JS — минимальный: появление блоков при скролле, переключение вкладок «География»,
  мобильное меню. Формы — визуальные, отправку не выполняют.

### Структура файлов
```
index.html              — главная
catalog.html            — каталог раздела (пример: «Промышленное оборудование»)
product.html            — карточка товара
about.html              — о компании
contacts.html           — контакты
assets/styles.css       — общий стиль
assets/main.js          — общий скрипт (reveal, вкладки, мобильное меню)
assets/img/             — изображения (на демо-этапе — внешние заглушки)
```

## 5. Страницы

### 5.1 Главная (`index.html`) — ГОТОВА
Блоки: верхняя плашка (город, график, ссылки) → header (логотип, кнопка «Каталог», меню,
телефон, «Оставить заявку») → герой на тёмном фото с заголовком и двумя кнопками →
плашка-уведомление «работаем с юрлицами, поставка под заказ» → цифры (15 лет, 12 000+ позиций,
240+ контрактов, 60+ городов) → каталог из 8 карточек-разделов с фото → блок «Берём на себя
весь цикл» (4 шага) → стена логотипов производителей → «География поставок» (вкладки городов +
панель с адресом) → CTA-плашка «Пришлите спецификацию» → подвал с реквизитами.

### 5.2 Каталог раздела (`catalog.html`)
- Хлебные крошки: Главная / Каталог / <Раздел>.
- Заголовок раздела + краткое описание.
- Слева — навигация по всем разделам каталога (текущий подсвечен).
- Справа — сетка товаров: 12–20 демо-позиций. Карточка товара: фото (ч/б), название,
  артикул, «Цена по запросу», метка «Под заказ», кнопка «Запросить цену».
- Карточка кликается → переход на карточку товара.

### 5.3 Карточка товара (`product.html`)
- Хлебные крошки.
- Крупное фото товара (можно мини-галерею-заглушку).
- Название, артикул, краткое описание.
- Таблица характеристик (производитель, тип, основные параметры — демо-значения).
- Блок цены: «Цена по запросу», метка «Под заказ», кнопка «Запросить КП».
- Блок «Похожие товары» (3–4 карточки).

### 5.4 О компании (`about.html`)
- Заголовок и текст о ТД «Меридиан».
- Переиспользуем блоки главной: цифры, «весь цикл», производители, география.
- Акцент на доверии: опыт, работа с 44-ФЗ/223-ФЗ, документы.

### 5.5 Контакты (`contacts.html`)
- Реквизиты ООО (название, ИНН, ОГРН, адрес — заглушки), телефон, почта, график.
- Простая форма «Оставить заявку» — визуальная заглушка (имя, телефон, сообщение, кнопка);
  отправку не выполняет, либо открывает mailto.
- Заглушка карты (статичное изображение/блок).

## 7. Явно вне объёма (YAGNI)

Корзина, цены, онлайн-оплата; реальный сбор заявок и админка; база товаров, фильтры и поиск;
личный кабинет; интеграция с Telegram (канал блокируется в РФ); многоязычность; CMS.

## 8. Критерии готовности

- 5 страниц связаны ссылками и открываются локально без ошибок.
- Единый визуальный стиль на всех страницах, адаптив работает на мобильном.
- Сайт целостно «читается» как действующий каталог поставщика и пригоден для показа заказчику.
- Везде, где это уместно, явная пометка о демо-характере данных.
